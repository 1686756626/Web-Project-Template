# 🔴 GitHub Contents API 推送中文文件会变空

> 来源：Teaching-Dashboard（commit `ee02867`，文件变为 0 字节）
> 严重度：🔴 红线，绝对禁止

## 规则

**永远不要用 GitHub REST API（Contents API）推送含中文内容的文件。API 返回 200 但文件变空。**

## 正确做法

```bash
git clone https://token@github.com/user/repo.git /tmp/repo
cd /tmp/repo
# 修改文件
git add -A && git commit -m "message" && git push
```

## 根因

Contents API 的 base64 编码对中文内容处理不稳定。API 不报错，但内容丢失。

## 验证

推送后通过 API 或 raw URL 获取文件内容，确认非空再结束。
