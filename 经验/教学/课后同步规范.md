# 课后同步规范

> 来源：CXL、Teaching-Dashboard

## 规则

**教学不记录 = 没教。** 每次课后必须更新仓库。

## 具体做法

### 更新步骤

```
1. 更新学生档案 → 课堂表现、测试结果、错题
2. 更新教学进度 → 过关状态、下一步、薄弱点
3. 更新电话记录（如有通话）→ 作业完成情况、学校进度、关注点
4. 更新今日任务 → 打勾
5. git add -A && git commit -m "更新：{学生名} {内容摘要}" && git push
```

### 电话记录格式

追加到学生档案的"电话记录"区域：

```markdown
### W09 周五（2026-04-03）
- 作业完成情况：XXX
- 学校进度：XXX
- 关注点：XXX
```

### 课堂反馈格式

```markdown
## 上次课反馈
- 整体表现：XXX
- 本次讲了什么：XXX
- 哪些没讲完：XXX
- 下节课优先：XXX
- 遗留问题：XXX
```

## 注意

- 记录事实时不脑补。有歧义的表述先确认再写入
- commit message 要清晰具体，说明改了什么
