# 多科目教学系统结构

> 来源：CXL（高考冲刺教学系统，6 科 110+ 文件验证通过）

## 规则

纯文档教学系统。智能体读文件→执行教学→课后更新文件。零代码。

## 标准目录

```
repo/
├── README.md              ← 一句话接管 + 每日启动流程
├── 学生档案.md             ← 性格、成绩、注意事项
├── 日程表.md               ← 每周固定课表
├── 排课总表.md             ← 每天每时段排课（与手册对接）
├── 今日任务.md             ← 今天具体要做什么
├── 定时任务配置.md          ← cron 任务模板
├── 教学时间管理规范.md      ← 每个时段怎么管控
├── 课后同步规范.md          ← 课后更新仓库的标准流程
├── 各科达标分析.md          ← 考试结构、分值、得分目标
├── 学习进度.md             ← 六科进度一览
├── 教学反馈记录.md          ← 历史反馈
├── 数学/                   ← 各科目录
│   └── 教学实施手册.md
├── 语文/
│   └── 教学实施手册.md
├── 英语/
```

## 核心设计原则

1. **一句话接管**：新智能体 clone 后按 README 的启动流程即可开始教学
2. **按需读取**：不要一次全读，按优先级加载
3. **每日任务文件**：每天一个任务清单，做完打勾
4. **课后必更新**：教学不记录 = 没教

## 教学实施手册格式（每科）

```markdown
# XX科教学实施手册

## Day 1：主题名
- 教学目标：...
- 教学内容：...
- 练习题：...
- 测试题：...

## Day 2：...
```

## 注意

- README 中的「每日启动流程」是智能体的 SOP，必须写清楚
- 时间管理规范是铁律：每时段查 3 次时间
- cron 任务首次接管时一次性配置，之后自动运行
