# 浏览器缓存破坏方案

> 来源：Teaching-Dashboard 项目，多次白屏事故

## 问题

更新 `app.js` 后浏览器使用缓存的旧版本，导致功能异常或白屏。

## 解决方案

在 `index.html` 的 `<script>` 标签上加版本号参数：

```html
<script src="app.js?v=20260404l"></script>
```

每次部署时递增版本号：

```bash
# 自动递增（简单方式：日期+字母）
sed -i 's/app.js?v=[^"]*/app.js?v=20260404m/' index.html
```

## 踩坑

1. **版本号必须同步**：源码目录和部署目录的 `index.html` 版本号要一致
2. **`<script>` 标签不能被截断**：曾因 `index.html` 末尾 `<script src="app.js?v=xxx"` 缺少 `"></script>` 导致 app.js 从未加载，白屏
3. **nginx 配置配合**：静态文件缓存 + `must-revalidate`，版本号变更后浏览器会请求新文件

## nginx 配置参考

```nginx
location ~* \.(js|css)$ {
    expires 1h;
    add_header Cache-Control "public, must-revalidate";
}
```
