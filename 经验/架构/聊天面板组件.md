# AI 聊天面板组件

> 来源：Finance-Manager、Andy-Shihundan（两项目验证）

## 规则

右下角浮动按钮 + 展开聊天面板。自包含组件，不依赖 `app` 对象。

## 需要的后端

| 服务 | 端口 | 用途 |
|------|------|------|
| CoPaw API | 8088 | Agent 对话（SSE 流式） |
| chat-server | 3099 | 消息持久化（Node.js + SQLite） |

### chat-server API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/sessions` | 会话列表 |
| GET | `/messages?session_id=xxx` | 消息历史 |
| POST | `/messages` | 保存消息 `{session_id, role, content}` |
| POST | `/messages/batch` | 批量保存 |
| DELETE | `/sessions/:id` | 删除会话 |
| GET | `/health` | 健康检查 |

### Nginx 代理

```nginx
location /copaw-api/ {
    proxy_pass http://127.0.0.1:8088/api/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_read_timeout 86400s;
    proxy_buffering off;
}

location /chat-api/ {
    proxy_pass http://127.0.0.1:3099/;
}
```

### config.js 配置

```js
chat: {
    enabled: true,
    agentId: 'YOUR_AGENT_ID',
    apiBase: '/copaw-api',
    chatApiBase: '/chat-api',
    userId: 'visitor',
    placeholder: '输入消息...'
}
```

## 注意

- SSE 流式对话需要 `proxy_buffering off` 和长超时
- chat-server 的 SQLite 数据库（`chat.db`）不进 Git
- systemd 服务模板见 `chat-server/xxx.service.example`
