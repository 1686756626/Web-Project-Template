# 贡献规范

> 本文件定义其他项目如何向 Web-Project-Template 贡献经验和可复用资产。
> 智能体在项目中发现新的通用经验时，按本规范执行。

---

## 一、什么该贡献

| 类型 | 标准 | 举例 |
|------|------|------|
| 踩坑经验 | 在 ≥2 个项目中遇到，或踩过一次大坑 | GitHub API 中文文件变空 |
| 通用组件 | 可独立使用，不依赖特定项目业务逻辑 | Store 工具、登录验证 |
| 架构模式 | 多项目验证过的设计模式 | 纯前端 SPA 标准结构 |
| 流程规范 | 跨项目通用的操作流程 | 教学时间管理、课后同步 |

## 二、什么不该贡献

| 类型 | 原因 |
|------|------|
| 项目特定业务逻辑 | 如五池分流规则、学生收费标准 |
| 一次性的配置数据 | 如某个 nginx 配置的具体 IP |
| 尚未验证的做法 | 只在一个项目中用过且不稳定的方案 |

## 三、贡献流程

```
1. 在本项目发现通用经验或可复用代码
2. 判断是否符合"该贡献"标准
3. 确定目标位置：
   - 经验文档 → 经验/{分类}/xxx.md
   - 可复用代码 → components/xxx.js
   - 模板/脚手架 → 根目录文件
4. 按格式模板编写（见下）
5. 更新本项目的 knowledge-map.json（contributes 数组）
6. 更新 Web-Project-Template 的 knowledge-map.json（索引）
7. 两个仓库分别 commit + push
```

## 四、新项目初始化流程

> 创建任何新仓库时必须执行，缺一不可。

```
1. GitHub 创建仓库
2. clone 到本地
3. 写 AGENTS.md（含项目定位、技术栈、红线、联动章节）
4. 创建 knowledge-map.json（模板见下方）
5. 从 Web-Project-Template 复制需要的 components/ 或模板代码
   → 在 uses 数组中记录
6. 更新 Web-Project-Template/knowledge-map.json 的 projects 索引
7. 两边分别 commit + push
```

### knowledge-map.json 模板

```json
{
  "_说明": "本项目与 Web-Project-Template 的联动配置。智能体接手时先读本文件。",
  "_知识库": "https://github.com/1686756626/Web-Project-Template",
  "_贡献规范": "Web-Project-Template/贡献规范.md",
  "contributes": [],
  "uses": [],
  "pending": []
}
```

### AGENTS.md 联动章节模板

```markdown
## X、通用经验知识库联动

> 详细联动关系见 `knowledge-map.json`（机器可读）。

知识库：[Web-Project-Template](https://github.com/1686756626/Web-Project-Template)
贡献规范：`Web-Project-Template/贡献规范.md`

### 规则

- 发现新的通用经验或可复用代码时，按 `贡献规范.md` 提取到知识库
- 提取后更新本项目的 `knowledge-map.json`（contributes 数组）和知识库的 `knowledge-map.json`
```

## 五、经验文档格式模板

```markdown
# 标题（描述问题或技术点）

> 来源：{项目名}（{验证状态}）

## 规则
一句话核心规则。

## 具体做法
可复制的代码或步骤。

## 注意
容易踩的坑。
```

## 六、组件代码格式模板

```javascript
/**
 * xxx.js — 一句话说明
 * 来源：{项目名}
 * 
 * 用法：
 *   简要说明怎么用
 */
// 代码...
```

## 七、目录分类

| 目录 | 放什么 |
|------|--------|
| `经验/前端/` | CSS、JS 前端技术方案 |
| `经验/部署/` | 服务器、nginx、CI/CD、cron |
| `经验/GitHub/` | GitHub API、git 操作相关 |
| `经验/架构/` | 设计模式、数据结构、项目结构 |
| `经验/教学/` | 教学方法论、流程规范 |
| `经验/数据/` | 数据格式规范 |
| `components/` | 可独立复用的 JS 代码文件 |
| 根目录 | 新项目脚手架模板（app.js/config.js/utils.js 等） |
